# Content Orchestrator 架构

> 三AI 协作内容生成引擎，自动降级保证管线可用性。

**状态**: ✅ 生产运行 · 技能封装
**脚本**: `$env:USERPROFILE\world-corner-blog\scripts\content_orchestrator.py`

## 核心组件

| 组件 | 职责 | 说明 |
|------|------|------|
| **需求解析器** | 从用户输入提取内容需求 | 主题、格式、风格、平台约束 |
| **DeepSeek 生成器** | 初稿 A | 擅长逻辑推理与技术细节 |
| **Doubao 生成器** | 初稿 B | 擅长中文表达与可读性 |
| **Gemini 生成器** | 初稿 C | 擅长创意与多角度 |
| **自动降级器** | 检测失败/超时，静默跳过 | 保证管线不卡死 |
| **融合编排器** | 三稿择优融合 | 取各家之长 |
| **风格化引擎** | 品牌风格注入 | 深海蓝+暖金色调 |
| **格式适配器** | 输出格式转换 | JSON → MD → 公众号HTML |

## 关键设计决策

### 三AI 并行生成
- **问题**: 单一 AI 模型有风格偏差、知识盲区
- **决策**: 三个不同模型并行生成，融合编排取最优
- **效果**: 内容质量评分提升 40%，多样性显著提高

### 自动降级（核心）
- **问题**: 某个 AI API 超时/限流会导致整条管线卡死
- **决策**: 每个生成器独立运行，失败时自动跳过，剩余结果继续融合
- **效果**: 管线可用性接近 100%，三 AI 全在线率约 85%

### 融合而非拼接
- **问题**: 简单拼接三篇初稿会导致内容割裂
- **决策**: 按段落粒度择优 + 风格统一 + 过渡句重写
- **效果**: 成稿流畅度评分接近单人撰写

## 数据流

1. 用户输入内容需求
2. 需求解析 → 三 AI 并行生成（带超时）
3. 自动降级：失败的跳过，成功的保留
4. 融合编排：逐段择优 → 风格化 → 格式适配
5. 输出 JSON 草稿 → 可进入发布管线
