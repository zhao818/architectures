# 双仓模式：知识库 + 工具仓 + 发文管线 架构

> 建立一个双仓库内容生态系统，将私有知识沉淀（knowledge-hub）与公共工具分发（tool-shed）分离，并通过一条标准化发文管线将知识素材转化为多平台文章。核心目标是以「冷静的科技观察者」定位持续输出有数据支撑的深度内容，同时通过免费开源工具引流、付费教学变现。

## 核心组件

| 组件 | 类型 | 说明 |
|------|------|------|
| **knowledge-hub** | 私有 GitHub 仓库 | 8 大学科 46 篇知识条目，每篇三段式（一句话 + 核心要点 + 能支撑什么观点），带交叉引用。发文时的素材源头 |
| **tool-shed** | 公开 GitHub 仓库 | 免费工具脚本，按 search / video / publish / ai-workflow / office 五模块组织。用于引流和建立技术专业度 |
| **发文管线** | 流程规范（methods/publishing-pipeline.md） | 八步标准化流程：选题 → 搜素材 → 写初稿 → 加幽默调剂 → 外部审稿 → 修改定稿 → 推微信草稿 → 多平台分发 |
| **多平台分发** | 分发层 | 微信（公众号）、知乎、掘金、即刻四平台 |

## 数据流

1. **触发**：热点驱动或知识库驱动选定选题 → 触发发文管线
2. **素材采集**：从 knowledge-hub 提取相关知识条目，同时通过 Google / Gemini / DeepSeek / 豆包四路搜索补充外部素材
3. **内容生成**：写初稿 → 加入幽默调剂素材（WikiHow / XKCD / 9GAG / 糗百 / KnowYourMeme Top5）提升可读性
4. **质量管控**：外部审稿环节主动搜索反方观点，确保论点经得起质疑 → 修改定稿
5. **分发**：先推微信草稿（标题 ≤9 字、摘要 ≤17 字）→ 再同步分发到知乎 / 掘金 / 即刻
6. **闭环**：成文后知识条目可能被补充回 knowledge-hub，形成积累 |

## 关键设计决策

1. **私/公分离策略** — 知识库私有不公开，保护核心知识资产；工具仓免费开源，降低获客门槛。发文时从私有库提取素材，保证内容独特性
2. **八步管线中嵌入幽默调剂** — 在写稿与审稿之间插入幽默素材（Step 3.5），避免纯技术文章枯燥，提升传播性
3. **外部审稿要求反方观点搜索** — 主动寻找对立论据，提高文章论证严谨度，支撑「冷静的科技观察者」定位
4. **内容质量铁律** — 不发鸡汤、每篇必须有知识库引用 + 外部数据支撑，与市面泛科技文章形成差异化
5. **变现模式分层** — 工具免费（引流）+ 录播教学 ¥19.9（初级付费）+ 陪跑 ¥99/小时（高端咨询），三层转化漏斗
6. **平台选择保守** — 首发微信（私域），再分发知乎/掘金/即刻（公域），暂不涉及视频平台 |

## 优化方向

1. **管线自动化** — 当前八步流程手动操作较多，可逐步将素材搜索、幽默素材注入、反方观点搜索、多平台分发等环节 AI 化/脚本化
2. **知识库增量更新** — 建立「发文即补充」的自动机制，发文后自动将新提炼的知识点追加到 knowledge-hub 并更新交叉引用
3. **知识库质量度量** — 引入知识条目使用频次统计、引用有效性检查，清理长期未被引用的条目
4. **平台分发差异化** — 当前多平台同内容分发，可优化各平台内容格式（标题长度、图文比例、互动引导）以适配不同用户群
5. **SEO 策略** — tool-shed 公开仓库可做 GitHub README 优化 + 技术博客索引，提升搜索引擎引流效果
6. **跨模块工具编排** — 将 tool-shed 五模块与发文管线深度整合，例如 publish 模块直接调用 search 和 ai-workflow 模块，减少切换开销
7. **用户反馈闭环** — 增加评论/咨询数据回溯到知识库的机制，将用户常见问题沉淀为新的知识条目 |
