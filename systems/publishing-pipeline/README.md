## 核心组件

| 组件 | 职责 | 技术选型 |
|------|------|----------|
| **内容预处理** | 元数据解析、标签提取、摘要 | Python + frontmatter |
| **格式转换** | MD→HTML / MD→纯文本 | Python Markdown |
| **模板渲染** | 品牌风格注入 | WeChat V4 模板 |
| **分发执行器** | 各平台 API 对接 | requests + Playwright |
| **PipelineTracker** | 管线可视化与状态追踪 | Python 上下文管理器 |
| **内容登记** | 发布记录持久化 | JSON 数据库 |
| **自动降级** | 单平台失败不影响其他平台 | try/except + 日志 |

## 平台适配矩阵

| 平台 | 格式 | 接口方式 | 特殊处理 |
|------|------|----------|----------|
| 微信公众号 | HTML | REST API | 标题≤9字、摘要≤17字、中文不乱码 |
| 知乎 | Markdown | API | 专栏+回答双模式 |
| 头条号 | HTML | API | 图片上传 |
| B站 | Markdown | API+Playwright | 专栏+动态 |
| 小红书 | 纯文本+图片 | Playwright | ShadowDOM 适配 |
| 抖音 | 视频描述 | Playwright | 禁止系列标签 |
| Twitter | 短文本 | API | 280 字截断+链接 |
| 视频号 | 视频+文案 | Playwright | WUJIE 框架适配 |

## 关键设计决策

### PipelineTracker 可视化
- **问题**: 多平台分发时间长（5-15分钟），用户不知道进度
- **决策**: `with PipelineTracker(steps, title)` 上下文管理器，实时显示进度
- **效果**: 用户可见每一步的执行状态，体验大幅提升

### 平台隔离
- **问题**: 一个平台失败卡住整条管线
- **决策**: 每个平台独立执行，失败只记日志不抛异常
- **效果**: 单平台失败不影响其他平台，整条管线可用性 > 99%

### 格式一次转多次适配
- **问题**: 每个平台各自做格式转换，重复且不一致
- **决策**: 预处理层统一转换为核心格式，适配层只做增量适配
- **效果**: 格式一致性提升，新增平台只需做适配层
