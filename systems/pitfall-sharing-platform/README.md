# 踩坑共享平台 (Pitfall Sharing Platform) 架构

> 将个人在工具/项目/工作流中踩过的坑（错误、故障、教训）结构化记录下来，公开共享，让他人通过搜索即可提前避坑。核心理念：**每个人踩过的坑，不该让别人再踩一遍。**

## 核心组件

## 1. claude-memory (GitHub 主仓库)
- **地址**: `github.com/zhao818/claude-memory` (public)
- **职责**: 唯一权威内容源，所有踩坑记录的存储、版本管理、索引入口
- **上层**: 通过手动 `git push` 驱动 Gitee 镜像同步

## 2. Gitee 镜像仓库
- **地址**: `gitee.com/Zhaotianbibg1/claude-memory`
- **职责**: 国内无障碍访问入口，解决 GFW 导致的 GitHub 不稳定性
- **关系**: claude-memory 的只读副本，单向同步，无独立内容

## 3. Hugo 静态站点 (待建)
- **URL**: `worldcorner.xyz/pitfalls/`
- **职责**: 博客站展示层，将 Markdown 踩坑文档渲染为 browsable 网页
- **内容源**: 直接从 claude-memory/pitfalls/ 目录读取

## 4. 标准化踩坑模板
- **前置元数据**: title, category(pitfalls), tags(工具/平台/问题类型), symptom, root_cause, fix, avoid, date
- **正文四段式**: 症状 -> 根因 -> 修复 -> 怎么避免
- **职责**: 保证每条踩坑结构一致，机器可读

## 5. Hugo Taxonomy 聚合系统
- **职责**: 按标签/工具/平台自动归类，搜索结果页直接展示 symptom + fix 摘要
- **关系**: 消费踩坑模板的 tags/metadata 字段

## 6. 已有内容素材库
- `global/feedback/` (7 条 feedback)、`global/tools/` (工具踩坑)、`patches/` (修复记录)、`projects/永德欠薪/` (项目级踩坑合集)
- **关系**: 待迁移/重写为统一踩坑模板格式

## 数据流

```
[AI 自动记录/人工撰写]
        │
        ▼
claude-memory（GitHub 主仓·public）      ← 内容在此创作、提交、版本管理
        │
        ├── 手动 git push  ──→  Gitee 镜像仓（国内可访问）
        │
        └── Hugo 构建 ──→  worldcorner.xyz/pitfalls/（静态 HTML 展示）
                                │
                        用户搜索 → Taxonomy 自动聚合
                                │
                        命中 → 展示 symptom + fix 一句话摘要
```

**关键环节**:
- **内容录入**: `pitfall-recorder` 技能触发时自动按模板写入 claude-memory，同时更新 MEMORY.md 索引和 TRIGGERS.md 路由
- **同步**: 当前为手动 `git push gitee master`，无自动触发
- **查询**: 用户通过 GitHub repo 搜索、Gitee 搜索、或（未来）Hugo 站点搜索引擎发现内容

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| **仓库策略** | claude-memory 即知识库，不另建独立仓库 | 避免内容碎片化，同一套版本管理+记忆系统 |
| **镜像选择** | Gitee 手动镜像 | GitHub 主仓 public，Gitee 解决国内访问障碍，手动 push 避免自动同步风险 |
| **模板设计** | YAML frontmatter + 四段式正文 | 机器可解析 metadata + 人类可读正文，Hugo taxonomy 直接消费 tags |
| **展示层** | Hugo 静态站点，不写 Web App | 零运维成本，Markdown → HTML 一步到位，天然 SEO 友好 |
| **内容范围** | 所有踩坑集中存，不分项目/工具/个人 | 跨项目复用，搜索时统一命中 |
| **纠偏** | 删除已建的独立 `pitfall-knowledge-base` 仓库 | 第一次误解后及时纠偏，回归 claude-memory 单仓方案 |

## 优化方向

| 方向 | 现状 | 目标 | 优先级 |
|------|------|------|--------|
| **Gitee 自动同步** | 手动 `git push gitee master` | Gitee 后台开 GitHub 镜像自动同步 | 高 |
| **内容整理** | 22→29 条已完成，但部分素材未改模板格式 | 首批 10 条按模板重写，统一仓库内所有踩坑格式 | 高 |
| **Hugo 站点上线** | 未建 | Hugo pitfalls section 模板 + `worldcorner.xyz/pitfalls/` 上线 | 中 |
| **搜索体验** | 依赖 GitHub/Gitee 原生搜索 | Taxonomy 聚合 + 摘要展示，减少查看全文成本 | 中 |
| **写入自动化** | 已部分通过 `pitfall-recorder` 技能 | 完整闭环：踩坑→自动分类→模板填充→提交→更新索引 | 低（已有基础） |
