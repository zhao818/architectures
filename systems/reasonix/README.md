# Reasonix 多智能体平台架构

> 多智能体协作与编排平台，支持动态角色分配、质量门控、自动降级。

**状态**: ✅ 生产运行 · Wave 6.0
**仓库**: `~/reasonix-server/`

## 架构概览

```
用户输入
    │
    ▼
┌─────────────────────────────────────┐
│           Orchestrator               │
│  （任务解析 → 智能体路由 → 调度）      │
└────┬──────┬──────┬──────┬────────────┘
     │      │      │      │
     ▼      ▼      ▼      ▼
┌─────┐ ┌─────┐ ┌─────┐ ┌──────────┐
│Scout│ │Judge│ │Role │ │Executor  │
│搜索 │ │评审 │ │Fix  │ │执行(独占)│
└─────┘ └─────┘ └─────┘ └──────────┘
  │       │       │         │
  └───────┴───────┴─────────┘
              │
              ▼
       ┌──────────┐
       │  输出     │
       │ 聚合+格式化│
       └──────────┘
```

## 核心组件

| 组件 | 职责 | 关键特性 |
|------|------|----------|
| **Orchestrator** | 任务解析、智能体路由、调度 | 动态角色分配、Wave 调度 |
| **Scout** | 信息搜索与收集 | 多源搜索、去重、摘要 |
| **Judge** | 质量评审与门控 | 多维评分、阈值控制 |
| **RoleFix** | 角色动态调整 | 任务→角色映射、上下文注入 |
| **Executor** | 最终执行（独占）| Wave 6.0 起独占执行权 |
| **AgentDropout** | 自动降级 | 智能体失效时自动切换备用 |

## 关键设计决策

### Wave 6.0: Executor 独占执行权
- **问题**: 多个智能体都能触发执行，导致职责不清、冲突
- **决策**: Executor 拥有最终执行独占权，其他智能体只做评审和辅助
- **效果**: 减少 60%+ 的冲突和重复执行

### AgentDropout 机制
- **问题**: 某个智能体 API 超时/失败时整个管线卡死
- **决策**: 每个智能体都有备用方案，失败时自动降级
- **效果**: 管线可用性从 92% 提升到 99.5%

### EARS 系统
- **问题**: 缺乏系统性错误分析与恢复机制
- **决策**: 错误捕获 → 分析 → 恢复 → 学习的闭环
- **效果**: 同一错误重复率下降 80%

## 数据流

1. 用户输入 → Orchestrator 解析意图
2. Scout 搜索相关上下文
3. Judge 评审搜索质量（是否需补充搜索）
4. RoleFix 确定执行角色与上下文
5. Executor 独占执行（Judge 可复审）
6. 输出聚合与格式化
