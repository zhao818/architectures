# AgentPrune 集成架构（Reasonix 多智能体系统） 架构

> 将 ICLR 2025 论文 AgentPrune（arXiv:2410.02506）集成到 Reasonix 多智能体系统中，通过分层剪枝策略减少冗余通信消耗，在维持推理质量的前提下实现额外的 Token 节省（目标 15%+ 额外，总计 50-60%）。

## 核心组件

## 分层剪枝管线（漏斗式协同）

### 1. AgentDropout（节点级剪枝）
- **职责**：在任务启动阶段决定哪些 Agent 节点被激活参与
- **位置**：漏斗第一层，输入任务后立即执行
- **输出**：激活的 Agent 子图（缩小了后续剪枝的搜索空间）

### 2. AgentPrune（边级剪枝）
- **职责**：在激活的 Agent 子图上剪枝冗余的通信边
- **位置**：漏斗第二层，在 AgentDropout 之后运行
- **核心技术**：图掩码模型（Graph Masking Model），学习预测哪些通信边是冗余的
- **输出**：优化的通信拓扑

### 3. EARS/ABSTAIN（行为级过滤）
- **职责**：过滤 Agent 的无效输出（"说的有没有用"）
- **位置**：漏斗第三层，通信执行之后
- **协同机制**：PerformanceTracker 监控各 Agent 的 ABSTAIN 率，超过 50% 时反馈给 AgentPrune 将其入边权重降低 50%

### 4. 协调器（Coordinator）
- **职责**：管理三层协同的优先级和动态调节
- **关键逻辑**：当 AgentPrune 剪枝率高时，自动降低 AgentDropout 率，防止过度剪枝

### 5. PerformanceTracker
- **职责**：实时监控 Token 消耗、推理质量、ABSTAIN 率等指标
- **触发角色**：触发在线微调（Token 突增 20%/质量下降 10%/新任务类型 30%+）

### 6. 训练管道
- **冷启动**：离线训练图掩码模型（200-500 条历史数据）
- **在线微调**：触发式更新，含经验回放(30%) + EWC 防遗忘

### 组件关系
```
任务输入 → AgentDropout → AgentPrune → 通信执行 → EARS/ABSTAIN → 最终输出
                ↑               ↑                          ↓
                └─── 协调器 ────┘          PerformanceTracker ──┘
                      ↑
              训练管道（冷启动+在线微调）
```

## 数据流

## 数据流

1. **任务输入** → 系统接收一个新任务
2. **AgentDropout 决策** → 输入任务特征 → AgentDropout 计算哪些 Agent 应激活 → 输出激活子图
3. **AgentPrune 图掩码** → 在激活子图上运行 Graph Masking Model → 输出二进制掩码（标记哪些通信边可剪）
4. **通信执行** → 按照掩码后的拓扑图进行 Agent 间通信
5. **EARS 行为过滤** → 每个 Agent 输出经过 EARS/ABSTAIN 过滤 → 无效输出被丢弃（ABSTAIN）
6. **PerformanceTracker 记录** → 记录 Token 消耗、推理质量、ABSTAIN 率等指标
7. **反馈闭环**：
   - ABSTAIN 率 > 50% → AgentPrune 降低该 Agent 入边权重
   - 协调器监测剪枝率 → 反向调节 AgentDropout 率
8. **在线微调触发** → 任一触发条件满足（Token / 质量 / 新任务类型）→ 启动图掩码模型微调

## 关键设计决策

## 关键设计决策

### 1. 分层而非单一剪枝
- **决策**：采用 AgentDropout（节点级）→ AgentPrune（边级）→ EARS（行为级）三层漏斗，而非单一面剪枝
- **理由**：三层在不同粒度工作，互补无冲突；AgentDropout 已剪掉部分冗余，AgentPrune 在其基础上进一步优化

### 2. 两阶段训练策略
- **决策**：冷启动（200-500 条历史数据离线训练）+ 在线微调（触发式）
- **触发条件**：Token 突增 20% / 推理质量下降 10% / 新任务类型占比 30%+
- **防遗忘**：经验回放(30%) + EWC（弹性权重巩固）

### 3. 动态协调机制
- **决策**：协调器反向调节 AgentDropout 和 AgentPrune 的剪枝强度
- **理由**：防止过度剪枝导致的推理质量崩溃；高剪枝率时自动降低节点丢弃率

### 4. POC 先行
- **决策**：先做 POC（3 个 Executor Agent + 50 个测试任务），成功后全量推广
- **成功标准**：Token 节省 ≥ 15%（额外）、质量下降 ≤ 5%、延迟增加 ≤ 10%、训练 ≤ 30 分钟
- **时间**：4.5 个工作日

### 5. 边际效益预期管理
- **决策**：目标额外 10-20% Token 节省（在 AgentDropout 基础上），总计 50-60%
- **理由**：AgentDropout 已消除部分冗余，AgentPrune 的边际收益递减，不追求不切实际的高比率

### 6. 风险优先级管理
- 训练数据不足 → 数据增强 + 迁移学习
- 在线微调遗忘 → 经验回放 30% + EWC
- 过度剪枝 → 任务复杂度自适应剪枝率（简单 50% / 中等 30% / 复杂 15%）
- 推理延迟 → 二值化掩码 + 缓存
- AgentDropout 冲突 → 协调器优先级控制

## 优化方向

## 优化方向

### 近期（POC 阶段）

1. **图掩码模型优化**：二值化掩码 + 缓存机制，降低推理延迟增幅（目标 ≤ 10%）
2. **数据增强**：解决训练数据不足问题，迁移学习补充冷启动数据
3. **复杂度自适应剪枝率**：简单任务 50% / 中等 30% / 复杂 15%，避免过度剪枝

### 中期（POC 通过后）

4. **Pareto 多目标优化**：Token 节省为主目标，推理质量为约束，参考历史经验 EXP-2024-091
5. **经验回放缓冲区**：30% 旧数据混合训练，防止灾难性遗忘（EXP-2024-088）
6. **渐进式学习率衰减**：配合经验回放，进一步提高在线微调稳定性

### 长期

7. **自动化触发阈值调优**：基于历史数据动态调整 Token/质量/新任务类型的触发阈值
8. **跨任务迁移学习**：AgentPrune 剪枝模式在不同类型任务间的迁移泛化
9. **端到端联合训练**：AgentDropout + AgentPrune + EARS 三层联合优化（替代当前独立训练）
