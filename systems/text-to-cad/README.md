# TextToCAD 架构

> 自然语言→CAD 代码生成系统，支持 FreeCAD 和 AutoCAD 双后端。

**状态**: ✅ 生产运行 · LangGraph 四工具
**仓库**: `~/langgraph-t2cad/`

## 架构概览

```
自然语言输入 (NL)
    │
    ▼
┌──────────────────────────────────────┐
│          LLM 解析层                   │
│  意图识别 → 参数抽取 → 约束解析       │
└──────────────────┬───────────────────┘
                   │
                   ▼
┌──────────────────────────────────────┐
│       LangGraph 编排引擎              │
│                                      │
│  ┌─────────┐  ┌───────────────┐      │
│  │tool_plan │─▶│tool_generate  │      │
│  │(规划)    │  │_code (生成)   │      │
│  └─────────┘  └───────┬───────┘      │
│                       │              │
│  ┌─────────┐  ┌───────┴───────┐      │
│  │tool_fix │◀─│tool_validate  │      │
│  │(修复)   │  │(验证)         │      │
│  └─────────┘  └───────────────┘      │
└──────────────────┬───────────────────┘
                   │
                   ▼
        ┌────────────────────┐
        │    CAD 执行层       │
        │  FreeCAD │ AutoCAD  │
        └────────────────────┘
```

## 核心组件

| 组件 | 职责 | 说明 |
|------|------|------|
| **NL Parser** | 自然语言→结构化参数 | LLM 解析，提取几何约束/参数 |
| **tool_plan** | 生成 CAD 操作序列 | 将 NL 指令拆分为可执行步骤 |
| **tool_generate_code** | 生成 CAD 脚本代码 | FreeCAD Python / AutoCAD LISP |
| **tool_validate** | 验证生成代码正确性 | 语法检查 + 逻辑校验 |
| **tool_fix** | 自动修复错误代码 | 基于错误信息重新生成 |

## 关键设计决策

### LangGraph 循环修复
- **问题**: 一次生成的 CAD 代码经常有语法/逻辑错误
- **决策**: 用 LangGraph 的循环机制，validate → fix 可迭代多轮
- **效果**: 生成成功率从 65% 提升到 92%

### FDE 方法论对齐
- 架构设计遵循 **Find → Decide → Execute** 三阶段
- tool_plan = Find，tool_generate = Decide，tool_validate+fix = Execute

### 双后端抽象
- FreeCAD 和 AutoCAD 通过统一接口层切换
- 生成时根据目标平台选择代码模板

## 数据流

1. 用户输入 NL 描述（如"画一个直径50的圆"）
2. tool_plan 拆解为操作序列
3. tool_generate_code 生成目标 CAD 代码
4. tool_validate 验证代码正确性
   - 通过 → 交付执行
   - 失败 → tool_fix 修复 → 再次验证（最多 3 轮）
5. CAD 后端执行代码，返回结果
