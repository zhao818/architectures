# 多智能体编排模式

> 跨系统的多智能体协作通用模式。

## 适用场景

- 多个 AI 模型/智能体需要协作完成一个任务
- 任务可分解为多个独立子任务
- 需要质量门控、自动降级、角色分工

## 核心模式

### 1. 编排器模式 (Orchestrator)

```
用户输入
    │
    ▼
┌──────────┐
│编排器     │──→ 子智能体 1
│(路由/调度) │──→ 子智能体 2
│           │──→ 子智能体 N
└──────────┘
    │
    ▼
  输出聚合
```

**使用**: Reasonix Orchestrator、LangGraph 编排

### 2. 评审门控模式 (Judge Gate)

```
智能体输出 → Judge 评审 → 通过 → 继续
                    │
                  不通过
                    │
                   ▼
                重试/修复
```

**使用**: Reasonix Judge、TextToCAD tool_validate

### 3. 并行择优模式 (Parallel + Best)

```
         ┌─────────────┐
         │  需求解析    │
         └──────┬──────┘
    ┌───────────┼───────────┐
    ▼           ▼           ▼
┌───────┐ ┌───────┐ ┌───────┐
│AI  A  │ │AI  B  │ │AI  C  │
│初稿   │ │初稿   │ │初稿   │
└───┬───┘ └───┬───┘ └───┬───┘
    └─────────┼─────────┘
              ▼
       ┌──────────┐
       │融合择优   │
       └──────────┘
```

**使用**: Content Orchestrator

### 4. 循环修复模式 (Loop Fix)

```
生成 → 验证 → 通过 → 结束
         │
       不通过
         │
         ▼
       修复 → 重新验证（最多 N 轮）
```

**使用**: TextToCAD tool_validate + tool_fix

### 5. 独占执行模式 (Exclusive Executor)

```
辅助智能体（搜索/评审/规划）
    │  (只有建议权)
    ▼
┌──────────┐
│执行器    │  ← 最终执行权唯一
│(独占)    │
└──────────┘
```

**使用**: Reasonix Wave 6.0 Executor

## 组合示例

Reasonix 同时使用了模式 1 + 2 + 5：
- Orchestrator 路由 (模式 1)
- Judge 评审门控 (模式 2)
- Executor 独占执行 (模式 5)
- AgentDropout = 模式 3 的降级变体

## 选择指南

| 场景 | 推荐模式 |
|------|----------|
| 单一任务多智能体协作 | 模式 1 + 2 |
| 内容生成需要多样性 | 模式 3 |
| 代码/脚本生成需要纠错 | 模式 4 |
| 需要明确权责分离 | 模式 5 |
