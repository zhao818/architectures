# ADR-001: 创建独立架构设计仓库

- **日期**: 2026-07-04
- **状态**: 已采纳
- **影响系统**: 所有系统

## 背景

之前系统架构设计分散在多个地方：
- claude-memory 的 session 记录里
- GitHub 仓库的 README 中
- 聊天记录里（一次性的）

没有一个统一的、可搜索、可演进的架构知识库。

## 选项

### 方案 A: 独立仓库 `architectures/`
- 优点：干净分离、独立演进、可放 diagram + ADR + 提案
- 缺点：多一个仓库

### 方案 B: 放入 `design-references/` 分支
- 优点：设计参考统一入口
- 缺点：架构设计 ≠ UI/UX 设计，混在一起语义混乱

### 方案 C: 放入 `claude-memory/`
- 优点：也是一种知识
- 缺点：claude-memory 是轻量记忆系统，不适合放大型架构图和提案

## 决策

选择方案 A，因为架构设计有独特生命周期（ADR 决策记录、版本演进、优化提案、外部参考），需要独立组织方式。

## 后果

- 正面：所有架构集中管理、可版本化、可搜索
- 正面：ADR 机制确保决策有记录可追溯
- 负面：需要维护多一个仓库（成本很低，因为内容更新频率不高）
